Ball Tracking using ROS 2 and OpenCV

The goal of this project is to create a ROS 2 node that detects a tennis ball in a live camera feed and tracks its position. The algorithm uses the OpenCV library for all computer vision tasks and publishes the processed video stream and the ball's coordinates to ROS 2 topics for other nodes to use.

The core computer vision logic follows these steps: Convert to HSV: The input camera image (in BGR format) is converted to the HSV (Hue, Saturation, Value) color space. The HSV space is generally better for color detection under varying lighting conditions compared to BGR or RGB. Color Filtering: A binary mask is created by keeping only the pixels that fall within a predefined HSV color range for a greenish-yellow tennis ball. Contour Detection: The outlines (contours) of all the white shapes in the binary mask are found. To reduce noise, the mask is first cleaned up using erosion and dilation operations. Process Contour: The algorithm identifies the largest contour, assuming it is the tennis ball. It then calculates the minimum enclosing circle and the precise center (centroid) of this shape. Visualize: A yellow circle is drawn around the detected ball, and a red dot is placed at its center on the original camera image, which is then published.

The primary weakness of this method is its reliance on static color filtering. It is vulnerable to significant changes in lighting conditions. If a shadow falls on the ball or the ambient light changes, the ball's HSV values can shift outside the predefined range, causing the detection to fail.
